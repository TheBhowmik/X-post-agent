🧠 WHAT THIS PROGRAM IS (1-line)

This is an MCP client that lets Gemini decide when to call tools, executes those tools via your MCP server, and feeds the results back to Gemini in the exact format it expects.

FULL SYSTEM (Bird’s-eye View)
You (terminal)
   ↓
MCP Client (this file)
   ↓
Gemini AI
   ↓ decides
Tool call needed?
   ↓
MCP Server (Express)
   ↓
Real-world API (Twitter, math)
   ↓
Result back to Gemini
   ↓
Final AI reply

SSEClientTransport is a transport layer class that:

Opens an SSE connection to the server

Receives server → client messages

Sends client → server messages via HTTP POST

Manages the MCP session lifecycle

It does not contain business logic—only communication logic.

9️⃣ Convert MCP tools → Gemini tool format

tools = mcpTools.tools.map((tool) => ({
  name: tool.name,
  description: tool.description,
  parameters: {
    type: tool.inputSchema.type,
    properties: tool.inputSchema.properties,
    required: tool.inputSchema.required,
  },
}));

🔥 CRITICAL

Gemini does not understand MCP schemas
You must translate them into functionDeclarations

What this does

It converts MCP tool schemas into Gemini-compatible function declarations.

Technical explanation

mcpTools.tools
→ array of tools returned by the MCP server

.map(...)
→ transforms each MCP tool definition into the format 
required by Gemini’s function-calling API

Each output object becomes a callable function schema for the model.

MCP tool schema ≠ Gemini schema

MCP tools use their own internal schema

Gemini expects:

{
  name,
  description,
  parameters: {
    type,
    properties,
    required
  }
}

This mapping is a schema adapter.

What happens after this

This tools array is passed to Gemini:

config: {
  tools: [{ functionDeclarations: tools }]
}

Which enables:

tool awareness

structured function calls

This maps MCP tool metadata into Gemini’s function declaration format so the model can discover and safely call backend tools.

let awaitingUserInput = true;

Purpose

Controls who should speak next:

true → wait for user

false → wait for Gemini

This prevents asking the user while a tool call is in progress.

USER INPUT PHASE
